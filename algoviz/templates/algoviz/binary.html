  • TIME COMPLEXITY = O(log n)

  • Binary search is a fast searching algorithm for finding a specific element in a sorted array or list.

  • The binary search method removes half of the remaining elements in each step, as opposed to linear search, which checks each element one at a time.

  • The target element and the middle element of the sorted list are compared first by the algorithm.

  • The search is effective if the target matches the middle element.

  • The search proceeds in the lower half of the list if the target is smaller, and in the upper half if it is larger.

  • Binary search significantly reduces the number of elements to be checked by repeatedly halving the search space; this results in a time complexity of O(log n), where n is the number of elements in the list.

  • Binary search is substantially faster than linear search due to its logarithmic time complexity, particularly for large datasets.

  • Sorting the input list is necessary for binary search, which can be problematic if the list is updated often.

  • But because of its efficiency, it works best in scenarios where search operations are frequent and the data is primarily static.
